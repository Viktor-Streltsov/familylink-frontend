# FamilyLink Frontend — Cursor Context

## О проекте

FamilyLink — мобильное приложение для улучшения коммуникации между родителями и детьми через AI-помощника. Пользователи создают семьи, отмечают своё настроение, описывают ситуации со своей стороны, получают AI-советы по общению.

Продукт разрабатывается для рынка Кыргызской Республики (позже — международного). Приложение с сильными требованиями к приватности: явные согласия на обработку эмоциональных данных, право на удаление, защита данных детей.

## Стек

- **Framework:** React 18 + Vite + TypeScript (strict mode)
- **Роутинг:** React Router v6
- **Styling:** Tailwind CSS
- **UI kit:** shadcn/ui (используем только этот kit, никаких MUI/Ant Design)
- **Формы:** React Hook Form + Zod для валидации
- **HTTP:** axios + TanStack Query (React Query)
- **State:** Zustand для глобального (auth), TanStack Query для серверного
- **Иконки:** lucide-react
- **Даты:** date-fns с локалью ru
- **Мобильная сборка:** Capacitor (iOS + Android из того же кода)
- **Deploy web:** Vercel

- Основные CTA (Create family, Submit, Get advice) — `--color-primary`

- Разрушительные (Delete, Revoke) — `--color-error` с уверенным подтверждением
- Никогда не используем чистый белый на фоне, только тёплый кремовый

## Правила кода

### Общие

- TypeScript strict mode, никаких `any`, только `unknown` где нужно
- Функциональные компоненты, никаких классов
- Именованные экспорты для компонентов, default — только для страниц/роутов
- Файлы компонентов — PascalCase (`FamilyCard.tsx`), утилиты — camelCase (`formatDate.ts`)

### API-запросы

- Все запросы через axios instance из `api/client.ts`
- JWT токен подставляется через axios interceptor автоматически
- Запросы оборачиваем в TanStack Query (`useQuery`, `useMutation`)
- Типы для запросов/ответов лежат рядом с API-функцией в том же файле

### Формы

- React Hook Form + Zod-схема
- Ошибки валидации показываем под полем
- Ошибки сервера показываем через toast (`sonner`)

### Auth

- Токен храним в localStorage под ключом `familylink_token`
- Информацию о текущем пользователе — в Zustand store
- Защищённые роуты — компонент `<ProtectedRoute>` в router.tsx
- При 401 от сервера — автоматический logout и редирект на `/login`

### Общий тон текстов в UI

- Обращение на «вы»
- Мягкий, поддерживающий тон
- Без канцелярита («Осуществите выбор» → «Выберите»)
- Без английских слов, где есть русский аналог
- Дисклеймеры про AI мелким шрифтом внизу карточки, но всегда

## Что НЕ делать

- Не использовать class components
- Не использовать Redux, MobX (только Zustand + TanStack Query)
- Не хранить чувствительные данные в state дольше, чем нужно
- Не показывать полный текст ошибки от сервера пользователю — только понятную формулировку
- Не делать свою систему компонентов — только shadcn/ui, если чего-то нет — добавляем через shadcn CLI

## Ключевые сценарии продукта

1. **Регистрация → создание семьи → приглашение супруга/ребёнка → отметка настроения**
2. **Создание ситуации → добавление своего описания → ожидание описания второй стороны → получение AI-совета**
3. **Управление согласиями → отзыв согласия → удаление всех эмоциональных данных**

## Safety-first правила

- При отображении safety-предупреждений AI (кризисные ситуации) — контакты помощи КР (111, 112) отображаются крупно и первыми
- В UI всегда есть быстрый доступ к «отозвать согласие» и «удалить мои данные» из настроек
- Никогда не скрывать факт AI-обработки за красивыми словами — всегда честно писать «AI даёт рекомендацию»
